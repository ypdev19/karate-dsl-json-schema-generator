# JSON to Karate DSL Schema Converter Project Overview

Client-side tool converting JSON payloads to Karate DSL validation schemas for QA/API testers. Fully themed, i18n-ready, offline-capable.

## Tech Stack

- Build: Vite | Framework: React 18 (Hooks) | UI: React Bootstrap 5 | React Router v6
- Editor: React-Ace + Ace-Builds | Icons: react-icons | Testing: AVA
- Contexts: GlobalContext (Theme/Lang) | Storage: localStorage | Lang: ES Modules

## Key Features

- **Core**: JSON → Karate Schema (recursive objects/arrays), required fields ("id, name, user.email"), type inference (#string/#number/#boolean/#[])
- **Upload + Clear All + DOWNLOAD**: 📁 Upload JSON, 🗑️ Clear All, 📥 Download Schema (output toolbar)
- **UI/UX**: Dual Ace Editors + Perfect toolbars, demo data, clipboard, toast animations, mobile-first responsive
- Convert button always visible (16px font). Show/Hide Snippet (conditional after conversion)
- **Theming/i18n**: Dark/Light toggle (persistent), EN/ES (localStorage persistent), real-time switching
- **Toolbar**: `[📁 Upload][🗑️ Clear All][Base Demo][Adv Demo]` - Perfect spacing/sizing
- Dev: Real-time JSON validation, zero-config, no backend/auth/routing
- **Generate behavior**: NEVER auto-clears required field (stays until Clear All)
- **ROUTING:** / (Home Converter) | /about | /privacy | /terms | /changelog
- NAVBAR: About/Changelog (+ Home button on other pages) | Footer: Terms/Privacy
- MD PAGES: Privacy/Terms load language-specific .md files (en/es) with ReactMarkdown
- CHANGELOG: Grouped by month/year with bullet lists + change count badges

- Editors 5-2-5 grid (lg+), stacked mobile
- **Toolbar**: Upload/Clear/Demos → Clear All disabled initially
- Convert always prominent
- Sticky footer responsive
- Routes: / (Converter) | /about | /privacy | /terms | /changelog
- Footer: Terms/Privacy (internal routing)

## Internal Conversion Logic

JSON: { "id": 101, "user": {"email": "..."}, "tags": ["admin"] }
↓ parse + traverse + type inference + required resolution
Karate: { "id": #number, "user": {"email": #string}, "tags": #[] }
↓ Snippet: Then match response == """{...}"""
